# Public Reproducibility

Anu Architecture is designed so a stranger can clone the repo, install
dependencies, set their own API keys, and reproduce every value.

## Three hard rules

### 1. Every data point traces to a public source

Every L## script must call a **public API** or document a **public download
URL**. Acceptable sources:

- FRED (free API key)
- BEA (free API key)
- BLS (free API key)
- World Bank, IMF, OECD (free APIs, no key needed)
- Government statistical agency websites with documented download URLs
- Published table/figure with documented page reference (digitization)

**Not acceptable**:
- Internal databases, proprietary data warehouses
- Files that exist only on the author's machine
- "I downloaded this from Bloomberg in 2018"

### 2. No synthetic data

If data is unavailable, the series is `"status": "DATA_UNAVAILABLE"` with an
empty/absent CSV. Never:

- `np.random` in a data construction script
- Linear-trend interpolation across a multi-year gap
- "Estimated" values without a documented analytical method
- Placeholders standing in for real values

Missing data is **a gap to resolve, not a hole to paper over**.

### 3. No proxies without justification

CPI is not PPI. Earnings is not compensation. Yield is not total return.

If the exact source is unavailable, document why and what proxy is used in
`project_registry.json` with `"proxy": true` and a justification. Every
substitution degrades faithfulness — make it explicit.

## API key management

- `config/api_keys.env` — your keys (ALWAYS gitignored)
- `config/api_keys.env.example` — placeholders + free-registration URLs
- L## scripts must gracefully degrade when keys are missing (warn, don't
  crash)
- `README.md` lists all required keys with registration links

## .gitignore must exclude

```
config/api_keys.env
data/raw-data/api/         # cached responses (regenerated by pipeline)
__pycache__/
*.pyc
```

## Audit

`anu-architecture audit` scans for:

- Hardcoded local paths (`/Users/`, `C:\Users\`, `D:\`)
- L## scripts without a `Public Source:` header
- `np.random` in data construction scripts (S/L/P/V/M paths)
- Missing proxy justifications when `"proxy": true`

Run it before every release. CI runs it on every PR.
